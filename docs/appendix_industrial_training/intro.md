# 附录 B：强化学习工程实践指南

> **本附录目标**：当你的模型从"单卡能跑"升级到"需要几十张 GPU 并行训练"时，这里的知识将成为刚需。我们从基础设施、分布式训练、评测体系到故障排查，覆盖工业级 RL 的全链路。

## 本附录结构

| 节 | 主题 | 核心问题 |
|---|---|---|
| [B.1 RL 基础设施纵览](./rl-infrastructure) | 从单机到 Agentic 的环境与采样架构 | 怎么把 RL 跑快、跑大？ |
| [B.2 分布式训练架构](./distributed-training) | 并行策略、混合精度、框架选型 | 怎么把大模型训练起来？ |
| [B.3 评测体系与 Badcase 分析](./evaluation-badcase) | Benchmark 选择、Badcase 归因、自动化闭环 | 怎么知道模型真的变好了？ |
| [B.4 训练监控与工业实战](./monitoring-practice) | 实时监控、故障排查、5 道综合练习 | 怎么在出问题时快速定位？ |

## 阅读建议

- **如果你在做 LLM 后训练**：按 B.1 → B.2 → B.3 → B.4 顺序阅读
- **如果你在做游戏/机器人 RL**：重点看 B.1 的环境并行化和 B.4 的故障排查
- **如果你在准备面试**：直接看 B.4 的工业实战练习
- **如果你只想查一个问题**：每节独立可查，按需跳转
